## What This Project Is

Agent SLAM is a live AI debate tournament. Two teams connect their AI agents to a central WebSocket server. The agents argue a topic (Finance, Marketing, or Ethics) automatically — zero human intervention once the match starts. A judging bot called The Oracle scores both agents and declares a winner.

We are building the agent that connects, listens, and argues better than every other team.

## Architecture (what we are building)

agent.py                  ← entry point, run this on match day
src/
config.py               ← loads .env, exposes all constants
state_machine.py        ← tracks live match state (turn, stance, phase, history)
ws_client.py            ← WebSocket connection, auto-reconnect, message routing
debate_engine.py        ← calls Claude API with web_search tool, returns argument
strategy.py             ← builds system + user prompt based on debate phase

### Data flow on each turn
1. ws_client receives `match-state` message → updates state_machine
2. state_machine detects it is our turn → triggers debate_engine
3. strategy.py builds prompt based on phase (opening/rebuttal/cross-exam/defense/closing)
4. debate_engine calls Claude API (claude-sonnet-4-5 + web_search tool)
5. Response trimmed to under 3000 chars → sent as `debate-message` JSON over WebSocket
6. state_machine records our message, increments counter, waits for next turn

## WebSocket Protocol (from official rulebook)

### Outgoing message format (what we send)
```json
{
  "type": "debate-message",
  "data": {
    "message": "Your argument text here"
  }
}
```

### Key incoming message types to handle
| Type | Action |
|---|---|
| `welcome` | Log, confirm connection |
| `match-state` | Update full state — turn, stance, topic, finishTime |
| `match-update` | Match started, extract finishTime |
| `debate-message` | Record opponent message, prepare our response |
| `match-paused` | Pause our timer, do not send |
| `match-resumed` | Resume, update finishTime |
| `match-finish` | Graceful shutdown |
| `previous-message` | Load conversation history on late join |
| `error` | Log reason, handle gracefully — never crash |
| `info` | Log only |

## Debate Strategy (how we win)

### Phase detection logic
| Phase | Trigger | Goal |
|---|---|---|
| `opening` | message_count == 0 | State position, 3 arguments, cite sources |
| `rebuttal_first` | message_count == 1 | Attack weakest opponent claim specifically |
| `cross_examination` | message_count 2–3 | Expose logical gaps, apply pressure |
| `defense` | message_count 4+ | Hold ground, deepen evidence |
| `closing` | seconds_remaining < 180 | Summarize wins, memorable final statement |

### Key winning advantages over other teams
1. Phase-aware prompting — most teams use a flat "debate" prompt
2. Closing argument detection — switch strategy in final 3 minutes
3. Source citation — web search gets real URLs, not hallucinated stats
4. Agility — system prompt explicitly instructs: read opponent's exact words, address them directly
5. Anti-hallucination — if not certain, argue from reasoning not fake data
